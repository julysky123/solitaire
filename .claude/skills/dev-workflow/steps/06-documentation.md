# 6단계: 문서 업데이트

## 목표

변경사항에 대한 문서를 필요한 경우에만 업데이트합니다.

## 원칙

**문서는 요청된 경우에만 생성/수정합니다.**

## 수행 절차

### 1. 문서 필요 여부 판단

| 상황 | 문서 필요 |
|------|-----------|
| 새 API 엔드포인트 추가 | O (API 문서) |
| 내부 함수 추가 | X |
| 설정 방법 변경 | O (README) |
| 버그 수정 | X |
| 사용자가 문서 요청 | O |

### 2. API 문서 업데이트 (해당되는 경우)

```markdown
## POST /api/users/login

사용자 로그인

### Request
- Body:
  - `email` (string, required): 이메일
  - `password` (string, required): 비밀번호

### Response
- 200: 성공
  - `token` (string): JWT 토큰
- 401: 인증 실패
- 400: 잘못된 요청
```

### 3. README 업데이트 (해당되는 경우)

```markdown
## 새로운 기능

### 다크모드 지원

설정에서 다크모드를 활성화할 수 있습니다.

```bash

THEME=dark npm start
```
```

## 문서 스타일

### DO (해야 할 것)

- 기존 문서 스타일 따르기
- 간결하고 명확하게
- 예시 코드 포함

### DON'T (하지 말 것)

- 새 README 파일 생성 (요청 없이)
- 과도한 문서화
- 자명한 코드에 주석 추가

## 체크리스트

- [ ] 문서가 정말 필요한가?
- [ ] 기존 문서 스타일과 일관되는가?
- [ ] API 변경이 문서에 반영되었는가?

## 주의사항

- **요청 시에만** - 문서 생성은 명시적 요청이 있을 때만
- **기존 스타일 유지** - 새로운 형식 도입 금지
- **필수 정보만** - 불필요한 정보 추가 금지
