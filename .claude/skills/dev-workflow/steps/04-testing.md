# 4단계: 테스트

## 목표

구현한 코드가 정상 동작하고, 기존 기능을 깨뜨리지 않았는지 확인합니다.

## 수행 절차

### 1. 기존 테스트 실행

먼저 기존 테스트가 모두 통과하는지 확인합니다.

```bash
# JavaScript/TypeScript
npm test

# Python
pytest

# Go
go test ./...

# Java
./gradlew test
# 또는
mvn test
```

### 2. 새 기능 테스트

구현한 기능이 요구사항대로 동작하는지 확인합니다.

#### 수동 테스트

```
- API라면: curl이나 Postman으로 요청
- UI라면: 브라우저에서 직접 확인
- CLI라면: 명령어 실행
```

#### 자동 테스트 추가 (필요시)

```
추가해야 할 때:
- 핵심 비즈니스 로직
- 엣지 케이스가 많은 경우
- 회귀 방지가 중요한 경우

추가하지 않아도 될 때:
- 단순한 CRUD
- 이미 충분한 테스트가 있는 경우
- 사용자가 요청하지 않은 경우
```

### 3. 테스트 실패 시

```
1. 에러 메시지 확인
2. 원인 파악
3. 코드 수정
4. 테스트 재실행
5. 통과할 때까지 반복
```

## 테스트 작성 가이드

### 테스트 구조 (AAA 패턴)

```javascript
test('설명', () => {
  // Arrange - 준비
  const input = ...;

  // Act - 실행
  const result = doSomething(input);

  // Assert - 검증
  expect(result).toBe(expected);
});
```

### 테스트 케이스 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| Happy Path | 정상 동작 | 유효한 입력으로 성공 |
| Edge Case | 경계 조건 | 빈 배열, 0, null |
| Error Case | 에러 상황 | 잘못된 입력, 네트워크 오류 |

## 체크리스트

- [ ] 기존 테스트가 모두 통과하는가?
- [ ] 새 기능이 요구사항대로 동작하는가?
- [ ] 엣지 케이스를 확인했는가?
- [ ] 에러 상황을 확인했는가?

## 주의사항

- 테스트 실패를 무시하지 않기
- 테스트 코드도 깔끔하게 작성
- 불필요한 테스트 추가 금지 (요청 시에만)
