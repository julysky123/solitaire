# 3단계: 개발

## 목표

요구사항을 코드로 구현합니다.

## 수행 절차

### 1. 태스크 분해

`TodoWrite` 도구로 구현할 태스크 목록을 생성합니다.

```
예시:
- [ ] UserService에 login 메서드 추가
- [ ] 로그인 API 엔드포인트 생성
- [ ] 입력 유효성 검사 추가
- [ ] 에러 응답 형식 정의
```

### 2. 기존 코드 분석

구현 전에 반드시 관련 코드를 읽습니다.

```
확인할 것:
- 기존 코드 스타일 (들여쓰기, 네이밍 등)
- 사용 중인 패턴 (예: Repository 패턴, DI 등)
- 관련 유틸리티/헬퍼 함수
- 타입 정의 (TypeScript의 경우)
```

### 3. 순차적 구현

각 태스크를 하나씩 구현합니다.

```
1. todo를 in_progress로 변경
2. 코드 구현
3. todo를 completed로 변경
4. 다음 태스크로 이동
```

### 4. 점진적 확인

큰 변경은 중간중간 동작 확인합니다.

## 코딩 원칙

### DO (해야 할 것)

- 기존 코드 스타일 따르기
- 최소한의 변경으로 요구사항 충족
- 의미 있는 변수/함수 이름 사용
- 필요한 경우에만 주석 추가

### DON'T (하지 말 것)

- 요청하지 않은 리팩토링
- 과도한 추상화
- 사용하지 않는 코드 추가
- console.log 남기기 (개발 중 제외)

## 보안 체크리스트

- [ ] SQL Injection 방지 (파라미터 바인딩)
- [ ] XSS 방지 (출력 이스케이프)
- [ ] 인증/인가 확인
- [ ] 민감 정보 로깅 금지
- [ ] 입력 유효성 검사

## 주의사항

- **Read 먼저** - 코드 수정 전에 반드시 기존 코드 읽기
- **작은 단위로** - 한 번에 너무 많이 변경하지 않기
- **테스트 가능하게** - 테스트하기 쉬운 구조로 작성
